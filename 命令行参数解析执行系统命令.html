Chapter 4 命令行参数解析/执行系统命令

4.1 Python 标准库：argparse 模块

4.1.1 1️⃣ 核心 (What)

argparse 模块是 Python 内置的 命令行参数解析工具，用于编写可通过命令行运行并带参数的程序。 它能自动生成 帮助文档 和 错误提示，让命令行工具更易用。

一句话总结：argparse 模块让 Python 脚本变成“命令行程序”。

4.1.2 2️⃣ 组成结构 (Dimension)

主要组件和功能：

  1. ArgumentParser 对象

    • 入口类，负责解析命令行参数。

  2. 参数定义方法

    • add_argument()：定义参数类型、名称、选项。

    • 参数类型：

      • 位置参数（必填）
      • 可选参数（带 - 或 -- 前缀）

  3. 常用配置

    • type：参数类型（int, float, str）。
    • default：默认值。
    • help：参数说明。
    • choices：限制可选值。
    • action：指定行为（如 store_true）。

  4. 解析与结果

    • parse_args()：返回包含参数的命名空间对象。

  5. 自动帮助

    • 自动生成 -h/--help 帮助说明。

4.1.3 3️⃣ 原理 (How it works)

  • argparse 读取命令行输入（来自 sys.argv）。
  • ArgumentParser 将输入与预定义的参数规则对比。
  • 如果匹配成功 → 返回一个 Namespace 对象（类似对象/字典）。
  • 如果不符合规则 → 自动打印错误信息和帮助提示。

4.1.4 4️⃣ 应用 (Usage)

  • 命令行工具开发：构建脚本的交互界面。
  • 批处理任务：让脚本支持不同参数运行。
  • 调试与实验：灵活切换输入参数。
  • 生产环境脚本：带 --help，让用户快速上手。

4.1.5 5️⃣ 举例 (Examples)

4.1.5.1 基本用法

import argparse

parser = argparse.ArgumentParser(description="示例程序")
parser.add_argument("filename", help="输入文件名")
parser.add_argument("-v", "--verbose", action="store_true", help="输出详细信息")

args = parser.parse_args()

print("文件:", args.filename)
if args.verbose:
    print("启用详细模式")

运行：

python script.py data.txt --verbose

输出：

文件: data.txt
启用详细模式

4.1.5.2 类型与默认值

parser.add_argument("--count", type=int, default=1, help="循环次数")

4.1.5.3 限制可选值

parser.add_argument("--mode", choices=["fast", "slow"], help="运行模式")

4.1.5.4 多值参数

parser.add_argument("--nums", nargs="+", type=int, help="输入多个数字")
# python script.py --nums 1 2 3

4.1.6 6️⃣ 辨析 (Misunderstanding)

  1. 误解 1：所有参数都必须是可选的。

    • 实际：位置参数是必填的，可选参数才有 -/-- 前缀。

  2. 误解 2：需要手动写帮助信息。

    • 实际：argparse 自动生成 -h/--help，只需写 help 描述。

  3. 误解 3：parse_args() 返回字典。

    • 实际：返回 Namespace 对象，可通过 args.attr 访问。

  4. 误解 4：报错信息需要手动处理。

    • 实际：argparse 遇到错误时会自动打印错误并退出。

✅ 总结： argparse 模块是 Python 的 命令行解析工具：

  • 定义参数 → add_argument()
  • 解析参数 → parse_args()
  • 自动帮助 → -h/--help

它比 sys.argv 更安全、更易维护，适合开发可复用的命令行工具。

4.2 Python 标准库：subprocess 模块

4.2.1 1️⃣ 核心 (What)

subprocess 模块是 Python 内置的 子进程管理工具，用于在程序中启动外部进程、执行系统命令，并与其交互（输入、输出、错误流）。 它是 os.system 和 os.spawn* 等旧接口的现代替代方案。

一句话总结：subprocess 模块让 Python 能像“外壳脚本”一样调用其他程序，但更安全、更灵活。

4.2.2 2️⃣ 组成结构 (Dimension)

常用接口和功能：

  1. 简单执行

    • subprocess.run()：运行命令，等待完成。
    • subprocess.call()：类似 run，返回退出码。
    • subprocess.check_output()：运行命令并返回输出。

  2. 进程控制

    • Popen 类：更底层的接口，支持实时交互。
    • 属性：.pid、.returncode。
    • 方法：.communicate()、.terminate()、.kill()。

  3. 输入输出管理

    • stdout：捕获标准输出。
    • stderr：捕获错误输出。
    • stdin：向进程传递输入。

  4. 参数选项

    • shell=True：通过 shell 执行命令（可能有安全风险）。
    • cwd：指定工作目录。
    • env：指定环境变量。
    • timeout：设置超时。

4.2.3 3️⃣ 原理 (How it works)

  • subprocess 底层通过 fork/exec（Unix） 或 CreateProcess（Windows） 启动新进程。
  • 父进程（Python）可以与子进程通过 管道 (pipe) 通信。
  • 进程执行完成后，返回退出码，0 表示成功，非 0 表示错误。
  • run() 封装了 Popen，简化了常见用法。

4.2.4 4️⃣ 应用 (Usage)

  • 系统命令执行：运行 shell 脚本、批处理命令。
  • 自动化任务：调用外部工具（如 ffmpeg, git, curl）。
  • 进程控制：启动后台服务、监控进程。
  • 数据管道：捕获外部命令输出并进一步处理。
  • 跨平台脚本：比 os.system 更安全、可控。

4.2.5 5️⃣ 举例 (Examples)

4.2.5.1 最简单的命令

import subprocess

subprocess.run(["echo", "Hello Subprocess"])

4.2.5.2 捕获输出

result = subprocess.run(["ls", "-l"], capture_output=True, text=True)
print(result.stdout)

4.2.5.3 获取命令输出

output = subprocess.check_output(["echo", "Python"], text=True)
print(output.strip())  # Python

4.2.5.4 使用 Popen

proc = subprocess.Popen(["grep", "python"], stdin=subprocess.PIPE, stdout=subprocess.PIPE, text=True)
out, err = proc.communicate("python is great\njava is ok")
print(out)  # python is great

4.2.5.5 设置工作目录和环境变量

subprocess.run(["ls"], cwd="/tmp")
subprocess.run(["printenv"], env={"MY_VAR": "123"})

4.2.5.6 超时控制

try:
    subprocess.run(["sleep", "5"], timeout=2)
except subprocess.TimeoutExpired:
    print("命令超时")

4.2.6 6️⃣ 辨析 (Misunderstanding)

  1. 误解 1：os.system 和 subprocess.run 没区别。

    • 实际：os.system 只返回退出码，无法获取输出；subprocess 更灵活安全。

  2. 误解 2：shell=True 总是必要的。

    • 实际：多数情况下推荐传列表形式 ["cmd", "arg1"]，shell=True 可能带来命令注入风险。

  3. 误解 3：check_output 总是安全的。

    • 实际：命令出错（非 0 退出）会抛出 CalledProcessError，需要捕获异常。

  4. 误解 4：Popen 和 run 作用相同。

    • 实际：run 是对 Popen 的封装，适合简单调用；Popen 更灵活，支持流式交互。

✅ 总结： subprocess 模块是 Python 调用外部程序的标准方式：

  • 简单调用 → run / check_output
  • 高级控制 → Popen
  • 捕获 I/O、设置环境变量、超时等功能非常实用。

它比 os.system 更强大、更安全，是现代 Python 项目推荐使用的进程调用工具。
