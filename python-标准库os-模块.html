Chapter 2 Python 标准库：os 模块

2.1 1️⃣ 核心 (What)

os 模块是 Python 内置的 操作系统接口，提供与文件系统、进程、环境变量等进行交互的功能。 它的地位就像一座“桥梁”：让 Python 程序能在不同操作系统（Windows、Linux、macOS）上调用底层功能，同时保持一定的 跨平台统一性。

2.2 2️⃣ 组成结构 (Dimension)

os 模块可以从以下维度来划分：

  1. 路径与文件操作

    • os.path 子模块：路径拼接、拆分、判断文件是否存在等。
    • 文件创建、删除：os.mkdir()、os.remove()。

  2. 目录操作

    • os.listdir()：列出目录内容。
    • os.chdir()：切换工作目录。
    • os.makedirs() / os.rmdir()：递归创建 / 删除目录。

  3. 环境变量

    • os.environ：读取和修改系统环境变量。

  4. 进程管理

    • os.system()：运行系统命令。
    • os.getpid()：获取当前进程 ID。
    • os.fork()（仅 Unix）：创建子进程。

  5. 跨平台抽象

    • os.name：当前系统 ('posix' / 'nt')。
    • os.sep：路径分隔符（Windows 是 \，Linux 是 /）。

2.3 3️⃣ 原理 (How it works)

  • os 模块本质上是 Python 标准库对底层系统调用的封装。
  • 在 C 语言层面，Python 解释器通过调用 POSIX API（Unix）或 Win32 API（Windows）实现功能。
  • os.path 是纯 Python 实现的辅助工具，进一步简化了路径处理。
  • 因为不同操作系统接口不完全一致，os 提供统一 API，Python 程序员只需要调用同样的函数，而不用关心底层系统差异。

2.4 4️⃣ 应用 (Usage)

常见应用场景：

  • 文件管理脚本：批量重命名、删除文件。
  • 自动化任务：运行系统命令，如调用 ffmpeg 转换视频。
  • 跨平台开发：通过 os.path.join() 确保路径在 Windows / Linux 都能正常运行。
  • 进程控制：获取当前 PID，用于守护进程或调试。
  • 读取环境变量：在部署时读取 API Key，而不是写死在代码中。

2.5 5️⃣ 举例 (Examples)

2.5.1 路径与文件操作

import os

# 拼接路径
path = os.path.join("folder", "file.txt")
print(path)  # 在 Linux 是 folder/file.txt, 在 Windows 是 folder\file.txt

# 判断文件是否存在
print(os.path.exists(path))

# 获取绝对路径
print(os.path.abspath("example.py"))

2.5.2 目录操作

# 创建目录
os.makedirs("test_dir", exist_ok=True)

# 列出目录内容
print(os.listdir("."))

# 删除目录
os.rmdir("test_dir")

2.5.3 环境变量

# 获取环境变量
print(os.environ.get("PATH"))

# 设置新的环境变量
os.environ["MY_VAR"] = "hello"
print(os.environ["MY_VAR"])

2.5.4 运行系统命令

# 执行系统命令（跨平台）
os.system("echo Hello OS")

2.6 6️⃣ 辨析 (Misunderstanding)

  1. 误解 1：os 模块和 shutil 一样。

    • 区别：os 提供底层接口（创建 / 删除 / 改名），shutil 提供更高层的文件操作（复制 / 移动）。

  2. 误解 2：路径拼接用字符串 '/' 就行。

    • 问题：跨平台时可能出错（Windows 不能直接用 /）。
    • 正确：用 os.path.join()。

  3. 误解 3：os.system() 返回命令的输出。

    • 实际：它返回的是命令的 退出状态码。
    • 如果想要输出，应该用 subprocess 模块。

  4. 误解 4：os.remove() 删除文件后不可恢复。

    • 实际：只是让文件系统释放引用，能否恢复取决于底层文件系统。

✅ 总结： os 模块是 Python 的 系统接口工具箱，核心价值是 跨平台封装。 写代码时：

  • 路径用 os.path
  • 文件用 os
  • 更复杂的操作用 shutil
  • 如果涉及子进程/输出，推荐用 subprocess 替代 os.system()。
